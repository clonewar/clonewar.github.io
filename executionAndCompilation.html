Javaプログラムのコンパイルと実行

プログラム実行までの流れ

JavaはSun Microsystems社が開発したプログラミング言語であり、
その開発環境の総称です。

1995年に発表された比較的新しいプログラミング言語ですが、
インターネットの普及とJavaの持つ優位性から開発者が多い言語でもあります。

Java言語の特徴について、軽く触れたいと思います。

オブジェクト指向言語

「オブジェクト指向」という概念はJavaだけでなく、
UMLやデザインパターンを知る上でも重要となります。

オブジェクトとは「もの」であり、「オブジェクト指向」とはソフトウェアの中で
現実世界の物事を形として「もの」で表現しようという概念です。

例えば「自動車」をオブジェクトとして表現するとこうなります。
「自動車」が持つ「速く走ることができる」という機能や「車輪が4つ付いている」などの
属性を定義することで「自動車」を表現できます。

オブジェクト指向で記載するメリットは
　・人間にとって解りやすいプログラムが書ける。
　・プログラムの再利用ができる。
　・ソフトウェア開発の効率が上がる。

ここでは「オブジェクト指向」を簡単に、やや強引に説明しましたが詳しく知りたい方は
下記の本を読んでみて下さい。
参考文献：図解Java流オブジェクト指向入門

Write Once, Run Anywhere(一度記述すれば、どこでも動く)

Javaは「Write Once, Run Anywhere」という構想で設計されています。
具体的には次のような特徴があります。

  • 一度作成すれば、どのプラットフォーム上でも動作する。
    • 「プラットフォーム」とはOSやハードウェアを組み合わせたコンピュータ環境
  • 一度作成すれば、オブジェクト指向によってプログラムを再利用できる。
  • 一度作成すれば、ネットワーク上のどのマシンからでも利用できる。
