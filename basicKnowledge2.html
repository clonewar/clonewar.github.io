Javaの基礎知識(2)

Javaの様々なプログラム

Javaのプログラムは「CUI」「GUI」「ネットワーク」などの
様々な環境を対象として作成できます。

作成したプログラムを、どの環境でも同じように動作させるため
Javaでは「Java仮想マシン」(Java Virtual Machine)と呼ばれる仕組みを利用して
この機能を実現しています。

「Java仮想マシン」はプログラム実行時に、
それぞれの環境で稼働するコードに変換しています。

このような機能を完備している為、
次に示すような環境で稼働するプログラムを、容易に作成できます。

CUI(Character User Interface)アプリケーション

Windows OSの「コマンドプロンプト」のような、
文字ベースのプログラムの実行環境をCUIと呼ぶ。

GUI(Graphical User Interface)アプリケーション

Windows OSの「デスクトップ上」のような、ウィンドウやアイコンなどを使った、
プログラムのグラフィカルな実行環境をGUIと呼ぶ。

Javaアプレット

アプレット(applet)は小さなアプリケーションという意味で、
Webブラウザ上で稼働するプログラムです。

HTMLファイル上で指定すると、
サーバからダウンロードされて、Webブラウザで実行される。
※アプレットは基本的に、ファイルの読み書きや実行が制限されています。

Javaサーブレット

サーブレット(servlet)は、Webサーバ上で稼働するプログラムです。
サーブレットの処理結果は、Webページとしてクライアントマシンに返されます。

CGIなどのプログラムとは異なり、一度呼び出されるとそのままメモリに常駐するため
高速な処理を行うことができます。

JSP(Java Server Pages)

JSPは断片的なJavaコードが埋め込まれたWebページです。
サーバ上でサーブレットに変換され、
処理結果がWebページとしてクライアントマシンに返されます。

Javaで記述されているため、Active Server Pageなどと異なり、
Webサーバのシステムに依存せず動作する事ができる。

JavaBeans

Javaで作成したプログラムを、
統合開発環境などで、視覚的に利用できる部品(BeanまたはBeans)
として利用するための仕組み。
