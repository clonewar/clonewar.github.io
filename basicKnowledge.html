Javaの基礎知識

Java言語の特徴

JavaはSun Microsystems社が開発したプログラミング言語であり、
その開発環境の総称です。

1995年に発表された比較的新しいプログラミング言語ですが、
インターネットの普及とJavaの持つ優位性から開発者が多い言語でもあります。

Java言語の特徴について、軽く触れたいと思います。

オブジェクト指向言語

「オブジェクト指向」という概念はJavaだけでなく、
UMLやデザインパターンを知る上でも重要となります。

オブジェクトとは「もの」であり、「オブジェクト指向」とはソフトウェアの中で
現実世界の物事を形として「もの」で表現しようという概念です。

例えば「自動車」をオブジェクトとして表現するとこうなります。
「自動車」が持つ「速く走ることができる」という機能や「車輪が4つ付いている」などの
属性を定義することで「自動車」を表現できます。

オブジェクト指向で記載するメリットは
　・人間にとって解りやすいプログラムが書ける。
　・プログラムの再利用ができる。
　・ソフトウェア開発の効率が上がる。

ここでは「オブジェクト指向」を簡単に、やや強引に説明しましたが詳しく知りたい方は
下記の本を読んでみて下さい。
参考文献：図解Java流オブジェクト指向入門

Write Once, Run Anywhere(一度記述すれば、どこでも動く)

Javaは「Write Once, Run Anywhere」という構想で設計されています。
具体的には次のような特徴があります。

  • 一度作成すれば、どのプラットフォーム上でも動作する。
    • 「プラットフォーム」とはOSやハードウェアを組み合わせたコンピュータ環境
  • 一度作成すれば、オブジェクト指向によってプログラムを再利用できる。
  • 一度作成すれば、ネットワーク上のどのマシンからでも利用できる。

Javaの様々なプログラム

Javaのプログラムは「CUI」「GUI」「ネットワーク」などの
様々な環境を対象として作成できます。

作成したプログラムを、どの環境でも同じように動作させるため
Javaでは「Java仮想マシン」(Java Virtual Machine)と呼ばれる仕組みを利用して
この機能を実現しています。

「Java仮想マシン」はプログラム実行時に、
それぞれの環境で稼働するコードに変換しています。

このような機能を完備している為、
次に示すような環境で稼働するプログラムを、容易に作成できます。

CUI(Character User Interface)アプリケーション

Windows OSの「コマンドプロンプト」のような、
文字ベースのプログラムの実行環境をCUIと呼ぶ。

GUI(Graphical User Interface)アプリケーション

Windows OSの「デスクトップ上」のような、ウィンドウやアイコンなどを使った、
プログラムのグラフィカルな実行環境をGUIと呼ぶ。

Javaアプレット

アプレット(applet)は小さなアプリケーションという意味で、
Webブラウザ上で稼働するプログラムです。

HTMLファイル上で指定すると、
サーバからダウンロードされて、Webブラウザで実行される。
※アプレットは基本的に、ファイルの読み書きや実行が制限されています。

Javaサーブレット

サーブレット(servlet)は、Webサーバ上で稼働するプログラムです。
サーブレットの処理結果は、Webページとしてクライアントマシンに返されます。

CGIなどのプログラムとは異なり、一度呼び出されるとそのままメモリに常駐するため
高速な処理を行うことができます。

JSP(Java Server Pages)

JSPは断片的なJavaコードが埋め込まれたWebページです。
サーバ上でサーブレットに変換され、
処理結果がWebページとしてクライアントマシンに返されます。

Javaで記述されているため、Active Server Pageなどと異なり、
Webサーバのシステムに依存せず動作する事ができる。

JavaBeans

Javaで作成したプログラムを、
統合開発環境などで、視覚的に利用できる部品(BeanまたはBeans)
として利用するための仕組み。
